Abstract

This page is a show case of how the famous model Bidirectional Encoder Representation from Transformers (BERT) model with two hidden layers on top of it makes desicion for a classfication task. The task is the popular Kaggle Competition Natural Language Processing with Disaster Tweets. This architecture helped in achiving a top 4% score on the leader dashboard. Using 10 examoles, that were chosen randomly, we will see that we can trust this model's predictions.

Introduction

After searching for a model that achives top 4% score for Kaggle's Natural Language Processing with Disaster Tweets competition. The dataset in this competition is a CSV file with an id column, which is a unique identifier for each tweet, text column, which contains the text of the tweet and other columns that were not used (along side the id column) in the training process. Further details about the dataset could be found in the data section on the competition's page. It is worth mentioning that throughtout this page we will be discussing the training dataset only.

The dataset contains 7613 tweets, some tweets contain hastages, mentions, and links. Tweets have a range of one word to 31 words, and the minmum number of charecter in a tweet is 7, while the maximum number is 157 charecters. Furthermore, they are distributed across two classes; real disaster, and not real disaster; as follow:


Bidirectional Encoder Representations from Transformers or BERT is a model designed to achieve state-of-the-art results when fine-tuned on several tasks, such as, classification, question answering, and language inference. When designing a language model, deciding the model does is crucial. In BERT's case, the model is trying to predict some masked tokens, as well as predicticting whether a sentence B is subsequent to sentence A. To better understand this model, one can refer to the orginal paper.
